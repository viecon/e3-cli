10LAN Conclusion 
●So the concept of LAN is for L2 or L3? 
●Both L2 and L3 have LAN concept 
○L2 LAN: Broadcast domain (Using MAC) 
■domain in which all nodes can reach each other by 
broadcast at the data link layer. 
○L3 LAN: Subnet (Using IP) 
●But more general definition of LAN: No need to be routed!

21Switched Virtual Interface (SVI) 
●Switch does not have a physical interface to which an IP address can be assigned. 
○IP is configured on a virtual interface called switched virtual interface (SVI) .
●Switched Virtual Interface (SVI) 
○A single SVI can only be mapped to a VLAN. 
○A SVI cannot be activated unless that VLAN associated with at least one  
active physical port. 
○SVI provides the Layer 3 processing for packets from all active physical ports 
associated with the VLAN. 
■routing packet from/to other SVI 
■no need physical router for inter-VLAN routing

32VLAN Numbering in Cisco IOS 
VLANs Range Usage 
0, 4095 Reserved Reserved for system use only. Not visible or usable. 
1 Normal Default Cisco VLAN. Cannot be deleted. Used for Control Plane 
Traffic (e.g., CDP , BPDU). 
2-1001 Normal Normal VLANs for Ethernet. Can be created, used, and deleted. 
1002-1005 Normal Default Cisco VLANs for FDDI and Token Ring. Cannot be 
deleted. 
1006-4094 Extended Extended VLANs for Ethernet only.

44Required Commands (Supplementary) - Dynamic Trunking 
●There is also dynamic mode, which can dynamically negotiate 
whether to use access or trunk mode 
○dynamic desirable: The port actively  attempts to switch the link 
to trunking mode. 
○dynamic auto: Passive  negotiation; only switches to trunk mode if 
the other switch actively requests it. 
(config)# ... omitted 
(config-if)# switchport mode dynamic  {desirable |auto }Keywords: "Cisco DTP", "dynamic trunking protocol".
