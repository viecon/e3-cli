1.Physical Layer –Transmits raw bit streams over a physical medium, like cables, 
voltages, and connectors
2.Data Link Layer –Provides node-to-node data transfer and error 
detection/correction across the physical link
3.Network Layer –Determines the best physical path for data and handles logical 
addressing and routing (e.g., IP)
4.Transport Layer –Ensures reliable end-to-end communication , through 
segmentation, flow control, and error recovery (e.g., TCP/UDP)
5.Session Layer –Manages and synchronizes dialogs or connections between 
applications
6.Presentation Layer –Translates, encrypts, and compresses data to ensure proper 
formatand representation between systems.
7.Application Layer –Provides network services directly to end-user applications such 
as email, file transfer, and web browsing.

How NAT/NAPT Works
For outgoing datagrams: Source NAT (SNAT)
Replaces (Sender IP , Port #) in Source Fields with (NAT IP, NAT Port # ),
Remember mapping (in NAT translation table ): 
(NAT IP, NAT port # ) (SenderIP address, Port # ) 
Sendsreplaced datagram to remote host 
Remote host replies with ( NAT IP, NAT Port # ) as Destination Transport 
address
For incoming datagrams: Destination NAT (DNAT)
Replace(NAT IP address ,NAT Port # ) in Destination Fields
with (SenderIP address, Port # ) stored in NAT table
Note:
SNATis performed after routing decision.
DNATis performed before routing decision4-51

Tables and Chains 
Filters are organized in a set of tables
Xtablesmanages tables and chains, and executes rules configured by Iptables
Tables: Containers for rules being performed on a packet
◦Filter: packet filtering, default table
◦NAT: NAT operation
◦Mangle: add tag on packet (for QoS or load distribution)
◦Raw: mainly for exemptions from connection tracking
Predefined Chains: checkpoints in the network stack where packets are processed
(Mapping to the five available Netfilterhooks)
◦PREROUTING : for packets before a routing decision is made.
◦INPUT: for packets destined to local sockets
◦FORWARD : for packets being routed through the machine.
◦OUTPUT: for locally-generated packets.
◦POSTROUTING : for packets about to go out after Routing decision has been made.
